## Overview

Every website you see has **3 different aspects**:

- **Skeleton (Blade/HTML):** Like blocks when writing in Notion
- **Skin (CSS):** Visual styling
- **Brain (PHP/Laravel):** Logic and backend

---

# PHP

**PHP (Hypertext Preprocessor)** is a server-side programming language used to build dynamic websites.
It is the **primary language** you should learn before moving on to Laravel.

---

## Getting Started

Install **XAMPP** or **Laragon** for your local environment. Create a file named: `index.php`.

Recommended editors:

- Visual Studio Code
- Cursor
- PhpStorm

Basic PHP example:

```php
<?php
echo "Hello World";

$name = "John";
echo "Hello, " . $name;
```

---

## Data Types & Variables

PHP is a _loosely typed_ language. Variables are prefixed with a `$` sign:

```php
$name   = "John";                    // String
$age    = 25;                        // Integer
$height = 1.75;                      // Float
$active = true;                      // Boolean
$hobbies = ["coding", "reading"];    // Array
```

# Laravel

**Laravel** is the most popular PHP framework today. Laravel simplifies the web development process by providing a comprehensive set of built-in tools.

Official documentation: https://laravel.com/docs

---

## Getting Started

First, install **Composer** (PHP's package manager):

```bash
# Install Laravel via Composer
composer create-project laravel/laravel my-project

# Navigate into the project folder
cd my-project

# Start the development server
php artisan serve
```

# Blade Templates (HTML Replacement)

**Blade** is Laravel's built-in template engine — but for PHP.
Blade files use the `.blade.php` extension:

```html
<!-- resources/views/welcome.blade.php -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>{{ $title }}</title>
  </head>
  <body>
    <h1>{{ $title }}</h1>
    <p>Hello, {{ $name }}!</p>
  </body>
</html>
```

---

## Blade Components (React Components Replacement)

Just like React Components, Blade also has reusable components:

```html
<!-- resources/views/components/button.blade.php -->
<button {{ $attributes->merge(['class' => 'btn btn-primary']) }}> {{ $slot }}</button>

<!-- How to use it in another view -->
<x-button class="btn-lg"> Login </x-button>
```

---

## Logic in Blade

```html
{{-- Conditionals --}} @if ($user->isAdmin())
<p>Welcome, Admin!</p>
@else
<p>Welcome, {{ $user->name }}</p>
@endif {{-- Looping --}} @foreach ($posts as $post)
<div>
  <h2>{{ $post->title }}</h2>
  <p>{{ $post->content }}</p>
</div>
@endforeach
```

# Routing

In Laravel, routing is defined in the `routes/web.php` file:

```php
// routes/web.php
Route::get('/', function () {
    return view('welcome');
});

Route::get('/user', [UserController::class, 'index']);
Route::get('/user/{id}', [UserController::class, 'show']);
Route::post('/user', [UserController::class, 'store']);
Route::put('/user/{id}', [UserController::class, 'update']);
Route::delete('/user/{id}', [UserController::class, 'destroy']);
```

---

## Dynamic Routes

```php
// Define route with a parameter
Route::get('/user/{id}', [UserController::class, 'show']);

// Inside the Controller
public function show($id)
{
    $user = User::findOrFail($id);
    return view('users.show', compact('user'));
}
```

# Controllers

Controllers in Laravel:

```php
// app/Http/Controllers/UserController.php
<?php
namespace App\Http\Controllers;

use App\Models\User;

class UserController extends Controller
{
    public function index()
    {
        $users = User::all();
        return view('users.index', compact('users'));
    }

    public function show($id)
    {
        $user = User::findOrFail($id);
        return view('users.show', compact('user'));
    }
}
```

Create a new Controller with Artisan:

```bash
php artisan make:controller UserController
```

# Eloquent ORM

**Eloquent** is Laravel's built-in ORM:

```php
// app/Models/User.php
<?php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    protected $fillable = ['name', 'email', 'password'];

    // Relationship: User has many Posts
    public function posts()
    {
        return $this->hasMany(Post::class);
    }
}
```

---

## Querying with Eloquent

```php
// Retrieve all records
$users = User::all();

// Find by ID
$user = User::find(1);

// Filter
$admins = User::where('role', 'admin')->get();

// Create a new record
User::create(['name' => 'John', 'email' => 'john@email.com']);

// Update
$user->update(['name' => 'Jane']);

// Delete
$user->delete();
```

# Migrations (Database Schema)

Migrations in Laravel replace **Prisma Schema**. They are used to create and modify database table structures:

```bash
# Create a new migration
php artisan make:migration create_posts_table

# Run migrations
php artisan migrate

# Rollback
php artisan migrate:rollback
```

```php
// database/migrations/xxxx_create_posts_table.php
public function up(): void
{
    Schema::create('posts', function (Blueprint $table) {
        $table->id();
        $table->string('title');
        $table->text('content');
        $table->foreignId('user_id')->constrained();
        $table->timestamps();
    });
}
```

# Authentication

Laravel provides **Breeze** or **Jetstream** as authentication starter kits, replacing Better Auth:

```bash
# Install Laravel Breeze (simple)
composer require laravel/breeze --dev
```

Features available after installation:

- Login with email/password
- New account registration
- Forgot password / password reset
- Social login via Laravel Socialite (Google, GitHub, etc.)

# Middleware (Route Protection)

Middleware in Laravel works the same as middleware in Next.js — it protects routes:

```php
// Protect a route with the auth middleware
Route::get('/dashboard', function () {
    return view('dashboard');
})->middleware('auth');

// Group of protected routes
Route::middleware(['auth'])->group(function () {
    Route::get('/profile', [ProfileController::class, 'show']);
    Route::get('/settings', [SettingController::class, 'index']);
});
```

# API with Laravel

Laravel can be used as a pure API backend. Use `routes/api.php` to define endpoints:

```php
// routes/api.php
Route::apiResource('users', UserController::class);

// Automatically generates the following routes:
// GET    /api/users          → index()
// POST   /api/users          → store()
// GET    /api/users/{id}     → show()
// PUT    /api/users/{id}     → update()
// DELETE /api/users/{id}     → destroy()
```

# Rendering in Laravel

## Server Side Rendering (Laravel Default)

Laravel uses SSR by default. The Controller fetches data and sends it to a Blade view:

```php
public function index()
{
    $posts = Post::with('user')->latest()->get();
    return view('posts.index', compact('posts')); // Data is passed to Blade
}
```

## Livewire (useState & CSR Replacement)

**Livewire** is a tool for adding interactivity without writing JavaScript:

```bash
php artisan make:livewire Counter
```

```php
// app/Livewire/Counter.php
class Counter extends Component
{
    public int $count = 0;

    public function increment()
    {
        $this->count++;
    }

    public function render()
    {
        return view('livewire.counter');
    }
}
```

```html
<!-- resources/views/livewire/counter.blade.php -->
<div>
  <h1>{{ $count }}</h1>
  <button wire:click="increment">Increment</button>
</div>
```
